# Diagramas

Los diagramas Mermaid del curso vivirán aquí. Deben explicar arquitectura,
responsabilidades o flujos; no decorar el capítulo.

## Diagramas actuales

- `01-modelos-de-servicio.mmd`: frontera de responsabilidades entre IaaS, PaaS,
  Serverless y SaaS.
- `02-computo.mmd`: flujo de decisión desde carga y restricciones hacia modo de
  cómputo.
- `03-almacenamiento.mmd`: flujo de decisión desde promesa del dato hacia forma
  de almacenamiento.
- `04-redes-y-vpc.mmd`: separación entre VPC, subredes, rutas y reglas de
  firewall.
- `05-identidad-y-accesos.mmd`: lectura desde principal y recurso hacia
  permisos mínimos, condiciones y duración.
- `06-servicios-manejados.mmd`: frontera entre responsabilidad delegada, estado
  durable y recuperación.
- `07-serverless.mmd`: flujo desde evento hacia función, retry, idempotencia y
  observabilidad.
- `08-costos-y-finops.mmd`: ruta desde gasto visible hacia dueño, unidad
  económica y presupuesto.
- `09-aws-en-la-practica.mmd`: traducción de conceptos cloud a servicios AWS
  con identidad, red, límites y costo.
- `10-gcp-en-la-practica.mmd`: traducción de capacidades cloud a servicios GCP
  con proyecto, identidad, labels y señales.
- `course-map.mmd`: mapa global del curso.
